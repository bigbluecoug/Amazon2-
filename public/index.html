Amazon credential guide
01 Register for Amazon Business API access.

Create or use an Amazon Business account, register in Amazon's Solution Provider Portal, submit the developer profile request, and wait for approval for the API roles you need.

02 Create an app client in SPP.

After approval, create an app client in the Solution Provider Portal. This app is what gives you Login With Amazon credentials for the integration.

03 Copy the Client ID.

Open the app credentials in SPP and copy the Client ID into this form. Keep the client secret private; it is only needed by the secure backend token exchange.

04 Authorize the app as an Amazon Business admin.

Use the authorization URL for your Amazon Business marketplace, sign in as the Business account admin, select Allow, and copy the short-lived OAuth code from the redirect URL.

05 Exchange the OAuth code for the refresh token.

Post the OAuth code, Client ID, client secret, and redirect URI to Amazon's LWA token endpoint. Save the returned refresh_token here; access tokens expire quickly, but the refresh token is used to request new access tokens.

Sandbox Testing shortcut.

For sandbox mode, create a sandbox app client, choose View sandbox credentials for the Client ID, then choose Create token in SPP to generate the sandbox refresh token.
